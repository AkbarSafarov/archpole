# Archpole — верстка сайта

---

## Стек

| Инструмент | Версия | Назначение |
|---|---|---|
| Gulp | 5 | Сборка, dev-сервер, вотчер |
| SCSS / Dart Sass | 1.72 | Стили с `@use`-модулями |
| Bootstrap | 5.3.8 | Сетка (`container-xl`, утилиты) |
| gulp-file-include | 2.3 | Подключение HTML-блоков (`@@include`) |
| Babel + preset-env | 7.24 | Транспиляция JS |
| BrowserSync | 3 | Live-reload в dev-режиме |

---

## Быстрый старт

```bash
npm install

# Dev-сервер с live-reload → build/
npm run dev

# Продакшн-сборка (минификация) → dist/
npm run build
```

## Структура проекта

```
src/
├── html/
│   ├── blocks/          # Переиспользуемые блоки (@@include)
│   │   ├── head.html    # <head> с мета-тегами
│   │   ├── header.html  # Шапка сайта
│   │   ├── footer.html  # Подвал + подключение всех JS
│   │   ├── product.html # Карточка товара (@@name, @@price, @@img)
│   │   └── ...
│   └── pages/           # Страницы → на выходе готовые HTML-файлы
├── styles/
│   ├── style.scss       # Точка входа — импортирует все модули
│   ├── components/      # Базовые компоненты (цвета, шрифты, кнопки, формы)
│   ├── blocks/          # Стили отдельных блоков / страниц
│   └── vendor/          # Bootstrap-конфигурация
├── js/                  # Скрипты (один файл = одна зона ответственности)
├── fonts/               # Шрифты
├── files/               # Изображения-заглушки (product1-5.jpg, dis1-3.jpg …)
└── libs/                # Сторонние библиотеки (Swiper, Bootstrap JS)
```

## Архитектура HTML

Блоки подключаются через `@@include` с передачей параметров:

```html
@@include('product.html', {
    "name": "Табурет «Минимализм»",
    "price": "7 800 ₽",
    "img": "files/product2.jpg"
})
```

Весь контент страниц пишется **напрямую в файл страницы** (`src/html/pages/*.html`) — промежуточные блоки-обёртки не создаются.

---

## Архитектура стилей

Единая точка входа `src/styles/style.scss` импортирует модули через `@use`:

```
components/colors      — CSS-переменные и цвета
components/fonts       — Объявление @font-face
components/buttons     — Базовые стили кнопок
components/forms       — Базовые стили форм

blocks/header          — Шапка и навигация
blocks/footer          — Подвал
blocks/product-item    — Карточка товара (листинг)
blocks/catalog-page    — Общие стили уровней каталога 3–6
blocks/catalog-5       — Главная страница каталога
blocks/catalog-6       — Листинг категории
blocks/pdp             — Карточка товара (детальная страница)
blocks/collection-page — Страница коллекции
...
```

**Соглашения по именованию:** BEM. Префикс блока отражает его назначение: `.cat5-`, `.cat6-`, `.pdp-`, `.coll-`, `.catalog-pg__` и т.д.

**Брейкпоинты** (через `max-width`):

| Название | Значение |
|---|---|
| Планшет | `max-width: 991px` |
| Мобайл | `max-width: 767px` |

## Скрипты JS

Все файлы подключаются глобально в `footer.html`. Каждый файл защищён проверкой наличия элемента (`if (el)`) — безопасно работает на страницах без нужного HTML.

| Файл | Назначение |
|---|---|
| `header.js` | Бургер-меню, поиск |
| `main.js` | Общая инициализация |
| `sliders-config.js` | Конфигурация Swiper-слайдеров |
| `catalog.js` | Логика каталога |
| `catalog-sort.js` | Сортировка, вкладки подкатегорий (cat6), теги коллекций (cat5) |
| `product-card.js` | Переключение фото по миниатюрам / точкам, аккордеон на PDP |
| `cart.js` | Корзина |
| `services.js` | Страница услуг |
| `feature.js` | Вспомогательные фичи |
